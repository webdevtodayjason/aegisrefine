// 1. QUOTE BEFORE PAYMENT

The customer gets a capped quote before work starts.

Aegis inspects the submitted dataset request, creates a signed quote token with a short expiry, and sends the buyer to Stripe Checkout only after the quote is accepted.

CODE
backend/app/services/quote_service.py
backend/app/routers/jobs.py
PROOF
Quote HMAC, expiry, user/job ownership checks, and Stripe Checkout metadata.
// 2. HERMES AGENT HANDOFF

The backend hands the paid job to Hermes, not to a hidden script.

The handoff payload includes job id, phase, source kind, quote, cap, checkout session id, existing spend tickets, and receipt context. Hermes returns structured JSON; the backend validates that response before advancing the job.

// 3. NEMOCLAW / OPENSHELL

The operator can run inside the live NemoClaw/OpenShell sandbox.

On the Dell R750, the sandbox is named aegis-hermes. In NemoClaw mode, the bridge wraps the exact same job payload with:

// 4. NVIDIA MODEL STACK

NVIDIA and Hermes each have a specific role.

Aegis-14B

Public LoRA fine-tune of Hermes 4 14B for dataset governance, route, risk, quality, and signing decisions.

Nemotron

Operations and safety model family for route, cap, spend, and content-safety judgment when configured.

NemoClaw/OpenShell

Sandboxed operator runtime for Hermes Agent, with provider credentials kept host-managed through the inference broker.

// 5. STRIPE MONEY RAILS

Checkout earns. Connect spends. The demo uses Stripe sandbox/test objects.

The hackathon demo uses Stripe sandbox mode deliberately: a real Checkout object starts the job, and a real Connect Transfer object can represent approved agent spend. The backend independently retrieves the Stripe object before recording spend as executed.

EARN RAIL
Stripe Checkout creates the paid job from the verified webhook.
SPEND RAIL
Hermes can initiate a Connect Transfer after cap approval.
VERIFY
Aegis checks transfer id, destination, amount <= cap, status, and Stripe livemode.
// 6. SIGNED DELIVERY

Every completed job closes with a signed receipt.

The customer receives the refined dataset plus an AAR certificate signed with Ed25519. The certificate records the job, route, model decisions, dataset stats, ledger, spend verification, and runtime receipt metadata.

// WHAT IS LIVE VS SANDBOX

Runtime is live; Stripe money is sandboxed for the demo.

Live

Aegis Refine app, Hermes Agent path, NemoClaw/OpenShell sandbox, NVIDIA inference route, public Aegis-14B model, dataset pipeline, Telegram receipts, and Ed25519 signing.

Sandbox/Test

Stripe Checkout and Stripe Connect objects are test-mode objects for the hackathon demo. Receipts store Stripe's own livemode field instead of hardcoding the claim.

Next hardening

Enable OpenShell GPU passthrough with NVIDIA Container Toolkit/CDI and cut Stripe to live-mode credentials when the business is ready for real customer money.
